# Platform Support Monitoring & Troubleshooting Lab

## Overview
This project simulates a real-world platform support environment using AWS EC2 and Ubuntu. It demonstrates system monitoring, log analysis, and troubleshooting of a web server.

## Tools Used
- AWS EC2 (Ubuntu)
- Apache Web Server
- Linux commands (top, systemctl, journalctl)
- Log files (access.log, error.log)

## Key Tasks
- Deployed Ubuntu server on AWS EC2
- Installed and configured Apache web server
- Monitored system performance using `top`
- Analyzed logs using `journalctl` and Apache logs
- Simulated service outages and performed troubleshooting
- Configured AWS security groups for network access

## Troubleshooting Example
- Stopped Apache service to simulate downtime
- Checked service status using `systemctl`
- Analyzed logs to identify issue
- Restarted service to restore functionality

## Outcome
Gained hands-on experience in platform support, system monitoring, and incident troubleshooting.
